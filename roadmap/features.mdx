## Development priorities

The OMNI team is focused on building features that:

- **Keep you in the terminal** — Every feature is designed to reduce context switching and keep your focus where it belongs
- **Work offline-first** — Your data lives locally. No cloud dependencies, no sync issues
- **Start instantly** — Commands should execute in milliseconds, not seconds
- **Stay minimal** — Each feature solves one problem well, without bloat or complexity

## Have a feature idea?

If you have suggestions for features that would make OMNI more useful, you can:

- Open an issue on GitHub to start a discussion
- Check the project roadmap to see what's already planned
- Contribute code if you want to build it yourself

The best feature ideas align with OMNI's philosophy: fast, distraction-free tools for terminal users.
